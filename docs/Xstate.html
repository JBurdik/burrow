XState v Burrow

Co to je, proč to tady máme a jak to reálně používáme.

Co je XState

XState je knihovna pro konečné stavové automaty (finite-state machines, FSM) a stavové diagramy. Místo hromady if/else a roztroušených status = 'running' popisuješ stavy a přechody deklarativně — co se smí stát v jakém stavu a co se stane při přechodu.

Verze 5 (kterou používáme) je kompletní přepis — žádné interpret(), žádné createMachine() bez setup(). Actors jsou first-class, typy jsou odvozeny automaticky.

Klíčový insight: FSM ti zakáže nemožné přechody. Nemůžeš přejít z idle rovnou do review — stroj to prostě ignoruje. Imperativní kód tenhle guardrail nemá.

Proč to v Burrow používáme

❌ Bez XState (původní kód)

Desítky míst kde se leaf.status = 'running' píše napříč Terminal.vue, RightPanel, stores. Přechod chybí? Bug. Zapomeneš vyčistit? Stuck stav. Pořadí eventů špatně? Race condition.

✅ S XState

Jeden stroj, jedna pravda. Přechody jsou explicitní — co není v diagramu, nemůže nastat. Logika je čitelná vizuálně. Debugger zobrazí historii eventů. Testy jsou triviální.

Konkrétní přínosy pro Burrow:

  • Stuck orange dot — nemůže nastat, pokud stroj nemá cestu jak v running zůstat navždy.
  • Premature done — guard isWatching je přímo v přechodu STOP → stroj zvolí done nebo review deterministicky.
  • Error persistence — error nemá žádný auto-timeout, jen MARK_SEEN nebo START. Nemůže se "ztratit".
  • Vizuální tooling — statelyai.com / XState VSCode extension kreslí diagram přímo z kódu.

Diagram stavů agenta v Burrow

idle ── START ──▶ running
running ── WAIT ──▶ waiting ── RESUME ──▶ running
running ── PERMISSION_REQUEST ──▶ permission ── RESUME ──▶ running
running ── STOP [watching] ──▶ done ── 4 s auto ──▶ idle
running ── STOP [!watching] ──▶ review ── MARK_SEEN ──▶ idle
running ── FAIL ──▶ error ── RETRY / MARK_SEEN ──▶ running / idle
Jakýkoliv stav ── INTERRUPT ──▶ idle

Kód: definice stroje (src/machines/agentStatus.ts)

import { setup, assign } from "xstate";
import { useMachine } from "@xstate/vue";

// 1. Typy eventů — co stroj akceptuje
export type AgentStatusEvent =
  | { type: "START" }
  | { type: "WAIT" }
  | { type: "RESUME" }
  | { type: "PERMISSION_REQUEST" }
  | { type: "STOP"; watching: boolean }   // guard závisí na payload
  | { type: "FAIL"; detail?: string }
  | { type: "MARK_SEEN" }
  | { type: "RETRY" }
  | { type: "INTERRUPT" };

// 2. Kontext — data nesená strojem (jen error detail)
interface AgentStatusContext { detail?: string }

// 3. setup() = typování + akce + guardy (vše typesafe)
export const agentStatusMachine = setup({
  types: {
    context: {} as AgentStatusContext,
    events:  {} as AgentStatusEvent,
  },
  actions: {
    // assign() vrátí parciální kontext — XState ho merguje
    clearError: assign({ detail: undefined }),
    setDetail:  assign({
      detail: ({ event }) => event.type === "FAIL" ? event.detail : undefined,
    }),
  },
  guards: {
    isWatching:  ({ event }) => event.type === "STOP" &&  event.watching,
    notWatching: ({ event }) => event.type === "STOP" && !event.watching,
  },
}).createMachine({
  id: "agentStatus",
  initial: "idle",
  context: { detail: undefined },

  states: {
    idle: {
      on: { START: "running" },
    },

    running: {
      on: {
        WAIT:               "waiting",
        PERMISSION_REQUEST: "permission",
        STOP: [
          { guard: "isWatching",  target: "done"   },
          {                         target: "review" },
        ],
        FAIL:      { target: "error", actions: "setDetail" },
        INTERRUPT: "idle",
      },
    },

    waiting: {
      on: {
        RESUME: "running",
        START:  "running",      // nový tah = reset čekání
        PERMISSION_REQUEST: "permission",
        STOP:      [{ guard: "isWatching", target: "done" }, { target: "review" }],
        FAIL:      { target: "error", actions: "setDetail" },
        INTERRUPT: "idle",
      },
    },

    permission: {
      on: {
        RESUME:    "running",
        START:     "running",
        STOP:      [{ guard: "isWatching", target: "done" }, { target: "review" }],
        FAIL:      { target: "error", actions: "setDetail" },
        INTERRUPT: "idle",
      },
    },

    done: {
      after: { 4000: "idle" },  // auto-clear po 4 s
      on: {
        MARK_SEEN: "idle",
        START:     { target: "running", actions: "clearError" },
      },
    },

    review: {
      on: {              // drží dokud uživatel tab neotevře
        MARK_SEEN: "idle",
        START:     { target: "running", actions: "clearError" },
      },
    },

    error: {
      on: {              // nikdy auto-clear — musí uživatel vidět
        RETRY:     { target: "running", actions: "clearError" },
        MARK_SEEN: { target: "idle",    actions: "clearError" },
        START:     { target: "running", actions: "clearError" },
      },
    },
  },
});

Kód: Vue composable

useAgentStatus() zabalí stroj pro Vue — vrátí reaktivní status a detail a funkci send(). Použij ji v každé komponentě, která potřebuje znát stav agenta. 

export function useAgentStatus(): AgentStatusComposable {
  // useMachine() = XState actor životní cyklus svázaný s Vue komponentou
  const { snapshot, send } = useMachine(agentStatusMachine);

  // snapshot.value.value = aktuální stav ("idle" | "running" | ...)
  const status = computed(() => snapshot.value.value as TermStatus);
  const detail = computed(() => snapshot.value.context.detail);

  return { status, detail, send, snapshot };
}

Praktické použití v Terminal.vue

Každý leaf (tab) dostane vlastní instanci stroje. Hooky posílají eventy místo přímého přiřazení.

// setup() v Terminal.vue
const agentMachines = new Map<number, AgentStatusComposable>();

function getMachine(leafId: number) {
  if (!agentMachines.has(leafId)) {
    agentMachines.set(leafId, useAgentStatus());
  }
  return agentMachines.get(leafId)!;
}

// Příchod stavu z burrow hook → event do stroje
function onAgentState(tab: Tab, leaf: Leaf, state: string) {
  const m = getMachine(leaf.id);
  switch (state) {
    case "running":
      m.send({ type: "START" }); break;
    case "waiting":
      m.send({ type: "WAIT" }); break;
    case "permission":
      m.send({ type: "PERMISSION_REQUEST" }); break;
    case "done":
      m.send({ type: "STOP", watching: isWatching(tab) }); break;
    case "error":
      m.send({ type: "FAIL", detail: leaf.statusDetail }); break;
  }
  // status se čte ze stroje — žádné leaf.status = "..." přímé přiřazení
}

// Čtení stavu pro UI
function leafStatus(leaf: Leaf): TermStatus {
  return agentMachines.get(leaf.id)?.status.value ?? "idle";
}

// Cleanup při zavření tabu
function closeTab(leaf: Leaf) {
  agentMachines.delete(leaf.id);  // actor se GC-uje, žádný leak
  // ... zbytek cleanup ...
}

Použití v template

<StatusDot

Praktické použití v Pinia store (Claude chaty)

Vue composable useMachine() je svázaná s komponentou. V Pinia storu potřebuješ nízkoúrovňové createActor(). 

import { createActor } from "xstate";
import { agentStatusMachine } from "@/machines/agentStatus";

// Při vytvoření chat session
function createSession() {
  const actor = createActor(agentStatusMachine).start();

  // reaktivita přes .subscribe() → Pinia state
  actor.subscribe((snapshot) => {
    session.status = snapshot.value as TermStatus;
    session.statusDetail = snapshot.context.detail;
  });

  return { ...session, _actor: actor };
}

// Příchod stavu z API streamu
function setStatus(sessionId: string, state: string) {
  const actor = sessions.get(sessionId)?._actor;
  if (!actor) return;
  if (state === "running")    actor.send({ type: "START" });
  if (state === "waiting")    actor.send({ type: "WAIT" });
  if (state === "done")       actor.send({ type: "STOP", watching: isWatching() });
}

// Cleanup
function destroySession(sessionId: string) {
  sessions.get(sessionId)?._actor.stop();
  sessions.delete(sessionId);
}

Guardy — podmíněné přechody

Guard je čistá funkce (context, event) → boolean. Stroj ho vyhodnotí při přechodu — pokud vrátí false, přechod se neprovede a zkusí se další varianta. 

// Stejný event STOP → různý cíl podle payload
STOP: [
  { guard: "isWatching",  target: "done"   },  // tab aktivní + focus = green flash
  {                         target: "review" },  // tab mimo focus = zelený puls dokud neotevřeš
]

V imperativním kódu bylo tohle if (isWatching()) leaf.status = 'done'; else leaf.status = 'review' schované v settleDone(). Se strojem je guard viditelný přímo v diagramu.

Časované přechody (after)

Stav done se po 4 sekundách automaticky vrátí do idle. Žádný setTimeout, žádný risk leaku — stroj timer spravuje sám a při destrukci ho zastaví. 

done: {
  after: { 4000: "idle" },  // ← XState spustí timer při vstupu do stavu
  on: {
    MARK_SEEN: "idle",         // ← manual cancel (klik na tab)
    START:     "running",       // ← nový tah cancelluje timer
  },
},

Kontext — data uvnitř stroje

Stroj nese minimální data (kontext) — v našem případě jen detail (typ API chyby jako "billing_error"). assign() je čistá akce která vrací parciální update kontextu. 

// Při FAIL eventu stroj uloží detail do kontextu...
FAIL: { target: "error", actions: "setDetail" }

// ...a clearError ho vymaže při přechodu z error stavu
RETRY: { target: "running", actions: "clearError" }

// V template pak detail jde přímo jako tooltip status dotu:
<StatusDot :title=`error: ${detail}` />

DevTools a vizuální tooling

Jedna z největších výhod XState je ekosystém nástrojů:

  • Stately Editor (stately.ai) — nakresli diagramy vizuálně, exportuj kód nebo importuj existující stroj.
  • XState VSCode extension — zvýraznění syntaxe, vizualizace přímo v editoru, skok na stavy.
  • XState Inspector (@xstate/inspect) — v dev modu zobrazí živé grafy všech běžících aktorů v prohlížeči.
  • Testování — machine.transition('running', { type: 'STOP', watching: false }) vrátí nový snapshot bez side-effectů. Čisté unit testy stavů bez mocků.
// Příklad unit testu (žádný Vue, žádný mount)
import { createActor } from "xstate";
import { agentStatusMachine } from "./agentStatus";

test("STOP bez watching → review", () => {
  const actor = createActor(agentStatusMachine).start();
  actor.send({ type: "START" });
  actor.send({ type: "STOP", watching: false });
  expect(actor.getSnapshot().value).toBe("review");  // ✅
  actor.stop();
});

test("STOP s watching → done, auto-idle po 4 s", async () => {
  vi.useFakeTimers();
  const actor = createActor(agentStatusMachine).start();
  actor.send({ type: "START" });
  actor.send({ type: "STOP", watching: true });
  expect(actor.getSnapshot().value).toBe("done");
  await vi.advanceTimersByTimeAsync(4001);
  expect(actor.getSnapshot().value).toBe("idle");  // ✅
  actor.stop();
});

Kdy XState nepoužívat

  • Jednoduchý boolean toggle (isOpen, isLoading) — ref(false) stačí.
  • Derived state bez side-effectů — computed() je lepší.
  • Async data fetching bez stavových přechodů — useAsyncData / TanStack Query.

XState svítí tam kde máš více jak 3 stavy, podmíněné přechody, timery, nebo kde buguje pořadí eventů. Status agent tabů je přesně tento případ.
