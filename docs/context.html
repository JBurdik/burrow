agentic-ide

Desktop IDE shell for running AI coding agents — Tauri + Vue 3 + Rust PTY

What it is

A native macOS desktop app (built with Tauri 2) designed as a minimal workspace shell around AI coding agents. It is not a code editor — it has no text editor pane. Instead it provides: a workspace picker, a real multi-tab split-pane terminal (PTY via Rust), a file explorer, a git panel, and a one-click agent launcher toolbar. The core idea: open a folder, click an agent button, agent runs in a terminal tab inside that workspace.

Architecture

Frontend
  • Vue 3 + Pinia + TypeScript
  • Vite build
  • xterm.js (PTY rendering)
  • @phosphor-icons/vue
  • No router — single screen
Tauri Bridge
  • invoke() for commands
  • listen() for PTY data events
  • Plugins: dialog, fs, os, shell
  • Capabilities: default.json
Rust Backend
  • portable-pty (real PTYs)
  • rusqlite (SQLite workspaces)
  • serde / serde_json
  • Per-PTY reader thread → emit events

Right Panel — Git

  • Branch display, refresh

  • Stage, unstage, stage-all
  • Diff viewer per file

Sub-agent delegation — burrow CLI

  • Agents run burrow spawn claude "…" to open a new tab and delegate work (fire-and-forget)
  • burrow collect [token…] pulls finished sub-agents' results without blocking (consumes each once); burrow sessions lists live tabs; burrow wait is the legacy blocking variant
  • Soft per-workspace cap (Settings → Agents → Max concurrent sub-agents, default 3) the /burrow skill respects
  • Transport = files in $BURROW_SESSION_DIR (no tty/socket); the app polls for requests
  • Subscription-safe: interactive only, never claude -p

Mission Control — tank-style task dashboard POC

  • Separate Tauri window (label mission) that runs headless claude PTYs as disposable tasks — spawn from a prompt, status dots, live-attach xterm, result capture, sequential queue
  • Reuses Burrow's native PTY + global hook server (no new wiring); 🚀 button in the ActivityBar opens it

What's New — v2.3.x

Fix: SessionStart no longer triggers "running"

Claude Code fires a SessionStart hook on every launch. Previously burrow hook mapped this to running, so fresh tabs opened with an orange/working dot before any prompt was sent. Fix: SessionStart now exit 0s in the burrow script and is removed from the registered hook event arrays in lib.rs. The burrow script is re-written to disk on every PTY spawn, so existing installs get the fix automatically.

Fix: Agent tab title tracks task description dynamically

Claude Code emits OSC 0;✳ Claude Code at startup, then updates the tab title to the current task description as the session progresses. Two fixes were needed:

  1. Idle revert blocked: after Stop, Claude resets its title to ✳ Claude Code. hookState tracks when a turn is done; onTitleChange blocks only that specific idle reset — other OSC titles pass through.
  2. Startup title preserved: Claude's startup OSC 0 arrives before the first 2 s poll sets foreground. pendingOscTitle now always buffers it. Shell precmd/preexec cwd noise (bare words like work, agentic-ide) cannot overwrite a meaningful title (one with spaces/Unicode) — priority: non-cwd > cwd.

Feat: OSC 7 live CWD tracking

Shells (zsh, bash, fish) emit OSC 7;file://host/path on every directory change. XTerm.vue now parses this sequence and emits a cwd event; Terminal.vue updates leaf.cwd immediately — no polling required. This keeps burrow spawn --cwd routing accurate when an agent cds into a sub-directory mid-session.

Feat: Double-click inline tab rename

Double-click any tab label in the Sidebar to rename it in-place. An inline <input> replaces the label (transparent background, accent-colored underline). Press Enter or blur to commit; Esc to cancel. Implemented via a store-based request (terminalTabs.rename(wsId, tabId, title)) — same pattern as activate/add/close. Terminal.vue watches the request and sets leaf.title on all leaves of the affected tab.

Feat: OSC 133 shell command boundary tracking

Shells (zsh, bash, fish, iTerm2-compatible) emit OSC 133;C on command-start and OSC 133;D on command-done. XTerm.vue now parses these markers and immediately emits busy(true/false) for non-agent sessions — no more waiting 2 s for the foreground poll to detect that npm install finished. No-op for agent tabs (hooks remain authoritative). Shell integration opt-in: users need the standard precmd/preexec hooks in their shell config (same hooks that power iTerm2/Warp shell integration).

Feat: Agent round counter

Each distinct "running" episode — i.e. each time the user sends a new prompt — increments a round counter on the leaf. The highest round across all leaves of a tab is published to the terminalTabs store and shown as a compact number (3) next to the status dot in the Sidebar, but only when > 1 and the tab is active. Lets you glance at the sidebar and know "this agent is on its third message exchange" without opening the terminal.

Feat: Permission request state + bell icon

Claude Code's PermissionRequest hook event now maps to a distinct "permission" agent state (previously collapsed into "waiting"). Surfaces as an amber pulsing dot (highest-priority status) + a bell icon in the Sidebar tab row — visually distinct from the blue "waiting for prompt" dot. The burrow script maps PermissionRequest → burrow status permission; terminalStatus.ts handles it like waiting (busy=true, plays waiting sound) but with its own TermStatus value so the UI can style it differently. Ctrl+C clears permission state (same as running/waiting).

Feat: OS window title syncs to active tab

The macOS window title bar now reflects the active tab: "task description — workspace name" when the tab has a distinct title, falling back to just the workspace name or "Burrow". Implemented as a watch in App.vue that calls getCurrentWindow().setTitle() via a lazy import() (so browser-only pnpm dev doesn't crash).

OSC Escape Sequences

SequenceDirectionMeaning
OSC 0 ; title BELPTY → appTab title + icon name (xterm). Claude emits this at startup (✳ Claude Code) and dynamically as the task description changes. Idle reset (✳ Claude Code) frozen after done; task titles pass through.
OSC 7 ; file://host/path BELPTY → appShell CWD hint — updates leaf.cwd live, no polling
OSC 133 ; C BELPTY → appShell: command started (non-agent tabs) → immediate busy=true, no poll wait
OSC 133 ; D BELPTY → appShell: command done → immediate busy=false
OSC 9998 ; running BELPTY → appAgent processing (UserPromptSubmit / PreToolUse / PostToolUse hook)
OSC 9998 ; waiting BELPTY → appAgent waiting for user input (Stop hook + idle Notification)
OSC 9998 ; permission BELPTY → appAgent needs permission decision (PermissionRequest hook) — amber bell
OSC 9998 ; done BELPTY → appAgent turn complete (Stop / SessionEnd hook)
OSC 9999 ; spawn ; … BELPTY → appLegacy direct-PTY spawn (latent parser — burrow CLI no longer emits this)

Note: OSC 9998 writes go via burrow hook → burrow status → HTTP server → pty-hook-{id} Tauri event → XTerm.vue. The OSC channel itself is only used internally by the old path; the HTTP path is primary.
