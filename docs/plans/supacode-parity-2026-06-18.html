Burrow ← Supacode parity plan

Co ukrást od konkurenta supabitapp/supacode a jak posunout Burrow na „command center“ pro AI agenty.
2026-06-18 · branch feat/supacode-parity · zdroj prozkoumán z gitu (Swift/macOS 26 · TCA · ghostty · Sparkle)

Supacode řeší stejný problém jako Burrow — orchestrace více coding agentů paralelně — ale v jiném stacku. Kód kopírovat nejde; bereme chování a architektonické vzory. Hlavní směr: agent umí ovládat celou aplikaci z CLI a sidebar sám vyplave to, co potřebuje pozornost.

🗣 Co Burrow nově umí (polopaticky)

Hotovo (už v PR #10)

  • Agent si teď umí sám ovládat appku. Doteď agent v terminálu uměl jen spustit dalšího agenta. Nově umí napsat do terminálu příkaz a Burrow ho poslechne: vypiš mi všechny projekty (burrow list-workspaces), vypiš taby (burrow list-tabs), přepni se na projekt 3 (burrow focus-workspace 3), přepni na tab 42 (burrow focus-tab 42), otevři nový tab a spusť v něm npm test (burrow new-tab --cmd "npm test"). Čili agent může sám proklikávat Burrow, jako bys to dělal ty myší.
  • „Needs Attention“ lišta nahoře v sidebaru. Když máš puštěných deset agentů v deseti projektech, nemusíš je obcházet a koukat, který na tebe čeká. Nahoře se sama objeví krátká lišta se VŠEMI taby, co potřebují tvou pozornost — agent čeká na odpověď, ptá se na povolení, spadl s chybou, nebo dodělal práci. Seřazené od nejnaléhavějšího (chyba → povolení → čeká → hotovo). Klikneš a skočíš rovnou tam. Když není co řešit, lišta zmizí.

Přijde vzápětí (Batch 2 — agenti zrovna píšou kód)

  • Tečky stavu už nebudou zasekané. Když agent spadne nebo ho zabiješ, neudělá „hotovo“ signál a oranžová tečka občas visí. Nově Burrow zná číslo procesu agenta a každou chvíli ověří, jestli ještě žije — když ne, tečku hned uklidí. Funguje i přes SSH.
  • Stav pull requestu uvidíš přímo v sidebaru. U každého worktree (větve) se ukáže odznáček s číslem PR a barvou — zelená = otevřený, šedá = draft, fialová = zmergovaný, červená = padají CI checky. Nemusíš na GitHub. (Funguje, když máš gh; jinak se prostě nic neukáže.)
  • Otevřeš i obyčejnou složku, nejen git repo. Doteď šel přidat jen git projekt. Nově hodíš do Burrow jakýkoli adresář a máš v něm terminály jako kdekoli jinde — git věci (větev, worktrees, diff panel) se prostě schovají.

🚧 Batch 2 — právě běží (každý ve vlastním worktree)

Izolované worktrees → žádné konflikty v lib.rs. Po doběhnutí: review → push → PR → cleanup (jako Batch 1).

běží  PID liveness sweep · feat/pid-liveness

Pid do burrow status POST → hook server ho forwardne → frontend periodicky ověří kill(pid,0) ekvivalent; mrtvý pid = tečka se settne okamžitě místo čekání na 3 prázdné polly. SSH gating (sweep jen u lokálního pid). Watchdog zůstává jako fallback.

Soubory: src-tauri/src/lib.rs · src-tauri/bin/burrow · src/components/XTerm.vue
běží  PR stav v sidebaru · feat/pr-status

Per-worktree PR přes gh pr view --json (number, state, isDraft, statusCheckRollup), cache v git store, refresh ~60s + na branch change. Badge: číslo + barevná tečka (zelená open / šedá draft / fialová merged / červená checks-fail). Degraduje bez gh (nic neukáže, nikdy nespadne).

Soubory: src/stores/git.ts · src/components/Sidebar.vue · src-tauri/src/lib.rs
běží  Folder (non-git) workspace · feat/folder-workspace

Jakýkoli adresář = workspace. is_git flag + zpětně kompatibilní migrace (ALTER TABLE ADD COLUMN default true). Pro folder skryje branch v titulku, New Worktree akci a git diff panel; terminály normálně. Add-workspace dialog bere libovolnou složku.

Soubory: src/stores/workspace.ts · src-tauri/src/lib.rs · src/components/Sidebar.vue · src/components/RightPanel.vue

📍 Roadmapa (po Batch 2)

vlajka Plná CLI ↔ deeplink parita

Dotáhnout Agenta A: každá akce má jak CLI formu, tak interní příkaz. Cíl — agent dokáže řídit celé UI (split panes, run script, archive worktree). Tohle je hlavní směr, co tě nadchnul.

další PR akce ve Spotlightu

Nad Batch-2 PR statusem: akce v paletě (open/ready/merge/rerun-CI). Branch watermarking — zahoď výsledek, když se branch mezitím změnila.

později Skripty (repo + global)

run / test / lint / format tlačítka per-workspace v toolbaru + Spotlightu + deeplinku. Per-repo config soubor (à la supacode.json), env BURROW_WORKTREE_PATH.

prověřit Per-řádek sidebar perf

Ověřit, že status update / agent tool-storm nererenderuje celou workspace listu. Pokud ano → memoizace per-řádek (supacode má dedikovaný per-row reducer + cached structure).

Transport — pozn.

Burrow zůstává u file requesty + HTTP hook server (odolnější, přežije daemon-reattach, žádné socket FD). Supacode socket neměníme. Ale stojí za přejmutí: víc env guardů v hooku (supacode vyžaduje SOCKET_PATH+WORKTREE_ID+TAB_ID+SURFACE_ID všechny set; Burrow gateuje jen na BURROW_PTY_ID) → méně false-positivů mimo Burrow.
