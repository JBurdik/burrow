Wrapper vs. bundling

cmux bundluje kopii Claude Code CLI do app bundle a injektuje shell funkci: 

# cmux přístup
claude () { "${_CMUX_CLAUDE_WRAPPER}" "$@" }

Burrow nebundluje binarku — místo toho injektuje env vars do každého PTY při jeho vytvoření (create_pty v lib.rs). Stejný efekt, nulový maintenance overhead. 

# lib.rs — create_pty, nastaveno automaticky
env.insert("CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS", "1");
env.insert("BURROW_PTY_ID",      id.to_string());
env.insert("BURROW_HOOK_PORT",   port.to_string());
env.insert("BURROW_HOME_DIR",    data_dir);
env.insert("BURROW_SESSION_DIR", session_dir);
// PATH prepend: <app-data>/bin/ (obsahuje burrow + tmux shim)
Výsledek: Každý claude spuštěný v Burrow terminálu automaticky vidí tmux shim, má agent teams zapnuté a má přístup ke všem burrow příkazům — bez jakékoliv konfigurace.

Tmux shim (src-tauri/bin/tmux)

Shell skript nainstalovaný vedle burrow v <app-data>/bin/. Přepsán při každém spuštění apky a každém create_pty. Adresář je prepend do PATH → tmux uvnitř Burrow PTY = shim.

Agent Teams

Claude Code zapíná multi-agent koordinaci přes CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1. Interně používá tmux pro správu oken. Se shimem v PATH agent teams fungují nativně v Burrow:

  • Každé tmux new-window otevře nový Burrow tab (přes burrow spawn)
  • Mapping @N → ptyId zapsán do $BURROW_HOME_DIR/tmux_wins/@N
  • Frontend zavolá register_tmux_win(winId, ptyId) ihned po vytvoření tabu

send-keys flow

tmux shim: send-keys -t @1 "hello" Enter
│ čte $BURROW_HOME_DIR/tmux_wins/@1 → ptyId=5
curl POST /write {ptyId:5, data:"hello\n"}
│ hook server
app.emit("pty-write-5", "hello\n")
│ Tauri event
XTerm.vue: invoke("write_pty", {id:5, data:[...]})
│ daemon socket
burrow-daemon: WritePty → PTY stdin
Timing: Mapping @N → ptyId je zapsán asynchronně přes register_tmux_win. Pokud send-keys přijde dříve než frontend zpracuje spawn request, soubor ještě neexistuje → shim tiše projde (exit 0). Agent teams obvykle čeká na inicializaci tabu, takže v praxi to nevadí.

burrow diff --last-turn

Zobrazí co se změnilo od začátku aktuálního agent tahu. Kombinuje CLI output + UI diff tab.

Jak funguje zachytávání HEAD

Globálně nainstalovaný Claude hook UserPromptSubmit volá burrow hook, který při tomto eventu: 

# v burrow hook subcommand (spouští se jako Claude hook)
git_head=$(git rev-parse HEAD 2>/dev/null)
printf '%s\n%s' "$git_head" "$PWD" \
  > "$BURROW_SESSION_DIR/last_head_${BURROW_PTY_ID}"

burrow diff --last-turn flow

burrow diff --last-turn
│ čte last_head_{ptyId} → old HEAD + CWD
git diff {old_head}
│ diff content (string)
curl POST /open-diff {ptyId, diff, title}
│ hook server emituje pty-open-diff-{id}
Terminal.vue: openDiffInTab(title, false, diff)
+ tiskne diff na stdout (pipeable) 
# Použití v agentovi
burrow diff --last-turn              # otevře diff tab + tiskne na stdout
burrow diff --last-turn | wc -l      # počet změněných řádků
burrow diff                          # prostý git diff (jen stdout)
UI tab: Diff tab se otevře jako nový leaf v aktivním workspace terminálu, stejně jako git diff taby spouštěné z pravého panelu. Funguje pouze uvnitř Burrow PTY (kde BURROW_PTY_ID je nastaven).

burrow set-status

Agenti mohou pushovat vlastní textový label do tab headeru vedle status dotu. 

burrow set-status "running tests..."    # zobrazí label v tab headeru
burrow set-status "deploying to staging"
burrow set-status                          # smaže label (žádné args)

Flow

# burrow script
curl POST /set-status {ptyId, text: "running tests..."}

# hook server
app.emit("pty-status-text-{id}", "running tests...")

# Terminal.vue listener
leaf.statusText = ev.payload  // nebo undefined při prázdném textu

Label se zobrazuje v tab baru za názvem tabu jako malý muted text. Vymaže se při dalším volání burrow set-status bez argumentů. Přetrvává přes status přechody (running/waiting/done) — agent ho musí sám smazat.

burrow trigger-flash

Krátká (600ms) CSS animace na tab labelu — vizuální signál "stalo se něco důležitého". Vhodné jako ping po skončení dlouhé operace, když agent nechce generovat notifikaci. 

burrow trigger-flash    # flashne aktivní tab na 600ms
/* Terminal.vue — CSS animace */
@keyframes tab-flash-anim {
  0%   { color: var(--accent); }
  50%  { color: var(--accent); opacity: 0.4; }
  100% { color: inherit; }
}
.tab-flash { animation: tab-flash-anim 0.6s ease-out; }
Implementace: flashingLeafs je reaktivní ref<Set<number>>. Flash přidá leafId do setu (new Set → trigger reaktivity), po 600ms ho odebere. Class tab-flash se aplikuje na tab label pokud je leafId v setu.

burrow top

Přehled všech živých Burrow PTY sessions — ptyId, CWD, titul foreground procesu. Čte data ze stejného daemon socketu jako burrow sessions. 

$ burrow top
PTY    CWD                                 TITLE
──────────────────────────────────────────────
1      /Users/me/project                   claude
2      /Users/me/project                   npm test
3      /Users/me/other-repo                zsh

Implementováno jako inline Python3 skript (stejný přístup jako burrow sessions, bez nových závislostí).

Hook server endpointy

Hook server (start_hook_server v lib.rs) nyní routuje podle URL. Přidané endpointy:

EndpointBodyAkce
POST /hook {ptyId, state} Existující — emituje pty-hook-{id} se stavem
POST /write {ptyId, data: string} Emituje pty-write-{id} → XTerm.vue zapíše do PTY
POST /set-status {ptyId, text: string} Emituje pty-status-text-{id} → leaf.statusText
POST /flash {ptyId} Emituje pty-flash-{id} → 600ms CSS animace
POST /open-diff {ptyId, diff, title} Emituje pty-open-diff-{id} → otevře diff tab
Port discovery: Hook server port je náhodný při každém startu. Zapsán do $BURROW_HOME_DIR/hook.port. Burrow skripty zkouší $BURROW_HOOK_PORT (baked do PTY env) i hook.port soubor — oba porty, první který odpoví vyhraje.

Frontend změny

TerminalSplitView.vue — Leaf interface

interface Leaf {
  // ... existující pole ...
  statusText?: string;  // set by `burrow set-status`
}

XTerm.vue — pty-write listener

unlistenWrite = await listen<string>(`pty-write-${props.ptyId}`, (ev) => {
  const bytes = Array.from(new TextEncoder().encode(ev.payload));
  invoke("write_pty", { id: props.ptyId, data: bytes });
});

Terminal.vue — per-leaf listenery

Každý leaf má sadu Tauri event listenerů registrovaných při vzniku, odstraněných při zániku. Tracked v leafUnlisteners: Map<number, UnlistenFn[]>. 

// Registrace při addTab / splitFocused / onMounted restore
registerLeafListeners(leaf.id)

// Odregistrace při closeTab / closePane / onBeforeUnmount
unregisterLeafListeners(leaf.id)

// Listenery na leaf (3 eventy)
pty-status-text-{id}  → leaf.statusText = payload
pty-flash-{id}        → flashingLeafs set + 600ms timeout
pty-open-diff-{id}    → openDiffInTab(title, false, diff)

SpawnRequest — tmux_win pole

// lib.rs
pub struct SpawnRequest {
  // ... existující ...
  pub tmux_win: String,  // @N, prázdný pokud ne tmux spawn
}

// Terminal.vue — po addTab
if (r.tmuxWin) {
  invoke("register_tmux_win", { winId: r.tmuxWin, ptyId: leaf.id });
}

Skill docs — co Claude vidí automaticky

install_agent_docs() se volá při každém startu apky. Přepisuje ~/.claude/skills/burrow/SKILL.md aktuální verzí. Claude v Burrow vidí nové příkazy bez jakékoliv manuální instalace.

Nové sekce v SKILL.md

## Status labels & visual signals
burrow set-status "running tests..."   # label v tab headeru
burrow set-status                       # smazání labelu
burrow trigger-flash                    # flashne tab

## Inspect what changed this turn
burrow diff --last-turn                 # git diff od začátku tahu

## Monitor all terminals
burrow top                              # tabulka živých PTY sessions

Co je automatické (žádná manuální konfigurace)

KdyCo se stane
Spuštění apky ensure_burrow_bin() zapíše burrow + tmux shim. install_agent_docs() přepíše skill docs. Hook server nastartuje s novými endpointy.
Nový PTY (create_pty) ensure_burrow_bin() znovu (= vždy aktuální binárky). CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1 v env. PATH prepend s bin/.
Nový tab (addTab) registerLeafListeners() — status-text, flash, open-diff listenery ready.
Spawn přes tmux shim burrow spawn --tmux-win @N zapíše tmux_win do request dir. Po claim Terminal.vue volá register_tmux_win(@N, ptyId).
UserPromptSubmit hook burrow hook uloží git HEAD do session dir → burrow diff --last-turn má referenci.
TL;DR: Rebuild + restart apky → vše funguje. Žádné claude settings, žádný manuální PATH, žádná instalace. Stačí otevřít terminál v Burrow.
