Burrow — Changelog

Historie verzí v češtině. Co se změnilo a proč.
v2.18.2 nejnovější červen 2026
Diagnostika — ACP v produkčním buildu
  • Vždy zapnutý log ACP adaptérů — spuštění adaptéru (resolved binárka, args, PATH), celý stderr, výsledek handshake i výstup turnů se zapisují do ~/Library/Application Support/com.agenticide.app/acp-debug.log. ACP se v zabaleném buildu chová jinak než v tauri:dev a selhání bylo jinak neviditelné; tohle ukáže přesnou příčinu.
v2.18.1 červen 2026
Opravy — ACP v produkčním buildu
  • ACP sezení se zaseklo v „thinking" bez modelu — handshake s ACP adaptérem neměl timeout: když adaptér v zabaleném buildu nenastartoval správně, čtení blokovalo navždy a chat zůstal viset bez modelu a bez chyby. Přidán 30s watchdog, který adaptér zabije a vrátí čitelnou chybu.
  • Viditelná příčina selhání — stderr adaptéru se dřív zahazoval; teď se posledních ~8 KB přilepí k chybové hlášce, takže je vidět skutečný důvod (chybí node, npm registry, auth…).
  • npx hang na install promptu — adaptéry spouštěné přes npx (claude-acp, codex) dostaly -y, aby se neptaly na potvrzení instalace; bez TTY četl prompt JSON-RPC vstup a sezení zamrzlo.
v2.18.0 červen 2026
ACP — nativní podpora víc agentů v chatu
  • ACP transport (Agent Client Protocol) — Claude chat umí mluvit nejen přes stream-json, ale i přes ACP. Nová vrstva AgentTransport + parser ACP session/update zpráv (acpParser.ts) překládá ACP události na stejný model bublin a stavů, jaký už chat používá.
  • Codex a Gemini jako chat agenti — vedle Claude lze v chatu spustit codex --acp a Gemini přes ACP. Agent switcher v hlavičce chatu cykluje claude → claude-acp → gemini → codex; ikony agentů přidány v agentIcons.ts.
  • Výchozí agent pro nový chat — nové nastavení defaultChatAgent (claude / claude-acp / gemini / codex) určuje, jaký agent se nastartuje při otevření nového chatu. Konfigurace přes ChatAgentConfig.vue v Settings.
  • Effort dropdown — u ACP agentů, které to podporují, lze v hlavičce vybrat úroveň reasoning effortu (vedle výběru modelu a permission módu).
Backend — řízení ACP procesů
  • Rust ACP lifecycle — nové Tauri příkazy acp_start / acp_send / acp_stop / acp_respond_permission spravují ACP agentní proces stejným modelem jako existující Claude proc. Restart (přepnutí agenta/módu, abort) prochází sjednoceným restartClaude(), který pro ACP korektně zastaví a znovu nastartuje adaptér a vyšle INTERRUPT, takže status tečka nezůstane viset.
  • ACP v burrow CLI — burrow zná ACP schopnosti agentů, takže delegace a status fungují i pro ACP sezení.
v2.17.0 červen 2026
Editor — LSP pro další jazyky + go-to-definition
  • Šest nových language serverů — editor teď rozpozná a napojí LSP pro Python (pyright-langserver), Go (gopls), C/C++ (clangd), Lua (lua-language-server) a shell (bash-language-server), navíc k dosavadnímu TypeScriptu a Rustu. Binárky se resolvují na Rust straně; chybějící server selže elegantně — soubor spadne na čisté zvýraznění syntaxe bez chyby.
  • Cmd/Ctrl-klik → přejít na definici — podržení ⌘ (resp. Ctrl) a klik na symbol skočí na jeho definici, jako ve VS Code. Aktivní jen když pro daný soubor běží LSP server (F12 funguje i nadále).
Claude chat — @-mention pilulky + slash mid-message
  • @-mention inline jako pilulka — soubory přidané přes @-autocomplete se teď vkládají přímo do textu na pozici kurzoru a v odeslané bublině se vykreslí jako kompaktní pilulka s ikonou souboru. Nahrazuje dřívější čipky pod inputem (museli se mazat zvlášť) — mention je teď součástí věty tam, kde ji napíšeš.
  • Slash příkazy uprostřed zprávy — nabídka /příkazů se spustí i když / není na začátku řádku, stačí mezera před ním. Doplnění vloží příkaz na pozici kurzoru, ne na začátek inputu.
Opravy
  • Zaseknutá status tečka po přepnutí nastavení — sjednocení restartu Claude procesu do jednoho helperu restartClaude() (sdílí ho přepnutí permission módu, modelu, profilu i abort). Restart teď vždy zahodí rozpracované permission/question/plan výzvy a vyšle INTERRUPT, takže status tečka už nezůstane viset na running/permission po přepnutí — proces, který je backoval, je pryč.
  • Odstraněn blikající kurzor u streamu — partial-cursor (blikající čárka u rozepsané odpovědi asistenta) byl odebrán; vlastní indikátor práce ho činil nadbytečným.

Chat UI — zásadní přepracování
  • Výstupy nástrojů inline — rozbalitelné karty nástrojů (Bash, Read, …) teď zobrazují i výstup. Ve sbaleném stavu se ukáže první řádek výstupu jako preview; rozbalením se zobrazí celý blok.
  • Fronta zpráv (Zed-style) — pokud je Claude zaneprázdněný, nová zpráva se zařadí do fronty místo čekání. Fronta se zobrazí jako sbalitelný panel nad inputem s počtem zpráv, možností odebrat nebo okamžitě odeslat kteroukoli z nich.
  • Indikátor práce — tři pulzující tečky s popisem aktuální aktivity (currentActivity) se zobrazí nad textarou, když Claude zpracovává odpověď.
  • Context usage bar — barevný proužek pod inputem ukazuje poměr spotřebovaných tokenů vůči limitu kontextového okna. Mění barvu při blížícím se limitu.
  • Cena tahu v toolbaru — cena session (v USD) se zobrazuje přímo v toolbaru vedle tlačítka Send místo ve stavovém řádku.
  • Přesun permission pickeru — výběr permission mode byl přesunut z hlavičky chatu do toolbaru (vedle ostatních nástrojů). Hlavička je teď čistší.
  • Čipky přiložených souborů — soubory přidané přes @-mention se zobrazí jako odstraňitelné čipky nad inputem.
Permission flow — šest módů + nový dialog
  • Šest permission módů — paleta rozšířena z původních tří na šest: Ask (výzva před každou akcí), Auto (Claude sám rozhoduje), Accept Edits (automaticky přijímá editace souborů), Plan Mode (jen plánuje, žádné akce), Don't Ask (provede bez dotazu, blokuje jen nebezpečné), Bypass (přeskočí vše). Každý mód má vlastní ikonu.
  • Rozevírací Allow menu — tlačítko Allow dostalo šipku s dropdownem: "Allow once", "Always allow [příkaz]…", "Always allow [nástroj]", "Deny". Granulárněji než dřív, kdy existovalo jen Allow / Always / Deny.
  • Rust: auto + dontAsk — claude_start v lib.rs nyní správně mapuje "auto" a "dontAsk" na parametr --permission-mode; dřív padaly na výchozí default.
Opravy
  • Sidebar dialogy oříznuté na sidebar — backdrop-filter na sidebaru vytvářel nový containing block, který ořezával position:fixed dialogy (výběr agenta, model picker…). Dialogy jsou teď teleportovány do <body>.
  • Chybějící import PhBell — PhBell nebyl importován v Sidebar.vue, Vue vypisovalo varování do konzole. Opraveno přidáním do importu.

v2.15.9 červen 2026
XState — stavový automat pro agentský stav
  • Jeden XState aktor per tab i per chat session — agentStatusMachine (XState v5) nahrazuje imperativní applyAgentEvent v Terminal.vue a přímé přiřazení session.status v claudeChats.ts. Každý leaf (tab) a každá chat session mají vlastní aktor uložený v Map; přechody (running → waiting → done → review…) jsou deklarativní.
  • Žádné race conditions — přechody přes XState stroj vylučují dosavadní třídění „kdo přijde poslední, vyhraje". Side-effecty (notifikace, zvuk, git refresh) se spouštějí jen z akcí stroje, ne volně po komponentě.
  • Perzistence přes restart — claudeChats.ts obnovuje aktory pro uložené session při init storu; aktory se stopují při odebrání session.
AgentTimeline — sbalitelný proužek, scrollovatelné řádky
  • Collapse toggle — záhlaví proužku s chevronem a počtem tahů umožňuje proužek sbalit/rozbalit. Sbaleno = jen tenký proužek pod tab barem.
  • Scrollovatelné tělo — 90 px panel s overflow-y: auto; všechny tahy jsou scrollovatelné, žádné ořezávání flexem.
  • Kompaktnější bary — výška řádku 26→14 px, osa 22→18 px, label 36→32 px — víc tahů na stejné ploše.
v2.15.7 červen 2026
Oprava: burrow collect vždy hlásil „still running"
  • Root cause — flag --settings /tmp/hooks.json šel v command stringu ZA pozičním argumentem (promptem). Novější Claude Code požaduje flagy před pozičním arg, takže ho tiše ignoroval → burrow capture TOKEN nikdy neproběhl → .done soubory nevznikaly → burrow collect hlásil „still running" navždy.
  • Oprava v XTerm.vue — --settings se teď vkládá hned za binárku přes replace(/^(\S+)/, '$1 flags') místo pouhého připojení na konec.
Manager — automatická notifikace o dokončení sub-agentů
  • Push notifikace do Manageru — když burrow capture TOKEN zapíše .done, pošle POST na hook server → Tauri event agent-done → ManagerBar ho zachytí a injektne zprávu do Manageru s 500 ms debounce pro batching: „Sub-agent t1 finished. Run: burrow collect t1". Manager pak collect zavolá sám bez toho, aby ho uživatel musel popostrčit.
AgentTimeline — SVG Gantt agentova tahu
  • Strip pod tab barem — aktivní agent tab zobrazí 120 px SVG Gantt s historií přechodů: kdy agent běžel (oranžová), čekal na vstup (modrá), čekal na permission (jantarová), hotovo (zelená). Ukáže se automaticky po prvním running eventu, zmizí po zavření tabu.
  • AgentHistory store (src/stores/agentHistory.ts) — sleduje TurnSegmenty per PTY, max 50 tahů zpět.
Git panel — konfigurovatelný auto-refresh
  • AutoRefreshButton vedle ruční refresh ikony — levý klik = toggle on/off, pravý klik = výběr intervalu (5 s / 10 s / 30 s / 60 s / vypnuto). Výchozí 30 s, uloženo do localStorage. Nahrazuje starý hardcoded setInterval.
  • Kde: Pravý panel → záložka Git → vedle ikony obnovení.
Pravý panel — responsivní layout (Container Query)
  • Panel sleduje vlastní šířku přes ResizeObserver. Pod 220 px: textové popisky záložek zmizí (jen ikony). Pod 320 px: inline diff panel se skryje.

v2.13.0 červen 2026
Manager — spodní lišta místo plovoucího okna
  • Manager je teď spodní lišta IDE — z plovoucí kartičky vpravo dole se stala vždy viditelná lišta dole, pod terminálem. Sbalená = tenký proužek s ikonou, status bodem a vstupním polem (napíšeš a odešleš rovnou Manageru). Rozbalená (klik na šipku nebo přetažení horního okraje) vyjede nahoru animovaný panel s historií zpráv. Výška panelu je tažením nastavitelná a pamatuje se.
  • Vstup zůstává v liště, rozbalená oblast jen pro historii — composer Manageru je trvale ve spodním proužku; rozbalený panel ukazuje jen minulé zprávy + povolovací/plán/dotaz gaty. Odeslání zprávy ze sbalené lišty nerozbaluje — úkol běží na pozadí a sleduješ ho přes status bod.
  • Per-workspace, i během běhu — každé zapojené repo drží vlastní mountovaný Manager chat (vzor jako u terminálů). Přepnutí workspace tak okamžitě ukáže Manager toho projektu — i když v původním běží tah. Žádné zabití claude procesu, běžící Manager dál streamuje na pozadí.
  • Mazlíčci běhají nad lištou — overlay zvířátek si zvedlo podlahu o výšku Manager lišty, takže pobíhají nad ní, ne za ní.
Manager — orchestrace & volba modelu
  • Manager jen orchestruje, nikdy nekóduje — systémový prompt nově tvrdě určuje roli: Manager nikdy nedělá práci sám. Jakýkoli úkol nad kódem (čtení, úpravy, fix, build/test, refactor) deleguje na spawnuté agenty a koordinuje je. Sám dělá jen orchestraci — píše zadání agentům, spravuje worktree/PR, sbírá výsledky a hlásí je uživateli.
  • Vlastní model Manageru — v liště přibyl přepínač modelu (Sonnet / Opus / Haiku, výchozí Sonnet) s vlastním klíčem nezávislým na ostatních chatech, takže Manager má svůj model bez ovlivnění běžných session.
  • Manager volí model spawnutému agentovi — Manager nově sám vybírá model pro každý delegovaný úkol přes claude --model <id>: Haiku na mechanickou práci, Sonnet jako výchozí, Opus na nejtěžší debugging/architekturu. burrow spawn flag rovnou přeposílá — žádná změna CLI nebyla potřeba.
  • Čitelný přepínač spawn-cíle — kryptická ikonka větev↔strom nahrazena popsaným dropdownem („Current branch" / „New worktree") s jednořádkovým popisem každého režimu.
Drobnosti
  • „Toggle Manager" ve Spotlightu — Manager se přepíná z command palety (⌘P) i klávesovou zkratkou ⌘J (nově v cheatsheetu ⌘/).
  • Konfigurovatelná pozice toastů — notifikace lze ukotvit do libovolného z 6 rohů/okrajů obrazovky (výchozí vlevo dole), směr odjezdu se přizpůsobí kotvě.

v2.11.1 červen 2026
Manager — opravený spawn agentů
  • Konec error: unknown option '--prompt' — systémový prompt Manageru neučil, jak předat úkol agentovi, takže si Manager vymyslel neexistující flag --prompt a spawn spadl. Nový prompt má výrazný blok CRITICAL SYNTAX: úkol se předává jako jeden citovaný poziční argument za claude (burrow spawn --cwd <dir> claude "celý úkol"), s explicitním zákazem --prompt / -p / --print.
Manager — přepínač worktree
  • Worktree per agent, nebo aktivní větev — v záhlaví Manageru přibyl přepínač (ikona větev ↔ strom). Výchozí: vypnuto = agenti se spouští přímo v aktivní větvi repa (žádný worktree). Zapnuto = každý spawnutý agent dostane vlastní izolovaný git worktree. Volba se pamatuje (localStorage) a systémový prompt Manageru ji reflektuje při každém tahu — Manager tak řídí spawn podle aktuální volby.

v2.10.0 červen 2026
Manager — plovoucí orchestrátor per projekt
  • Z Mission Control chatu je „Manager" — plovoucí kartička (kulaté tlačítko vpravo dole) je teď perzistentní per kořenové repo: drží stejnou session i když přepneš na jiný worktree téhož repa, místo aby byla pokaždé prázdná. Skrytá session se navíc nezobrazuje v seznamu chatů v postranním panelu.

Opravy & výkon
  • Konec beachballu při otevření chatu — claude_start i claude_get_account byly synchronní Tauri commandy běžící na hlavním vlákně; blokující fork/exec claude a 4s claude status tak na první otevření chatu zmrazily UI a roztočily kurzor. Nově jsou asynchronní, mimo hlavní vlákno.
  • Nové taby se nereattachují na zavřenou session — tab zavřený v minulém běhu nechal svoji PTY žít v daemonu a vypadl ze seznamu uložených; čítač pty-id pak mohl novému tabu přidělit to samé id a create_pty se připojil na starou session („objevil se starý terminál"). Dvě opravy: čítač se posouvá za každou session v daemonu (vč. orphanů, mimo MC range), a zavření tabu nově počká na kill_pty + uloží stav, takže orphani nevznikají.
  • Žádný falešný toast „dokončeno" při přepnutí režimu — přepnutí permission režimu (a abort) restartuje claude; teardown exit už nevyhodí notifikaci o dokončení tahu.
Drobnosti
  • Terminal pets — rozlučková exploze — zavření agentského tabu odpálí jeho zvířátko (rozletující se střípky, záblesk, „bye!"). Přibyla žirafa, krtek vyměněn za želvu.
