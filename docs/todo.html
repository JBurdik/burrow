Burrow — Feature Backlog

Features worth stealing from terax-ai, manaflow, Warp, iTerm2, xterm.js ecosystem. Sorted by effort within each section.

Quick Wins — hours, not days

quick xterm.js Ctrl+F in-terminal search

xterm.js ships a SearchAddon. Wire Ctrl+F → open a mini search bar above the terminal → highlight matches, prev/next navigation. Essential when Claude produces long output.

  • Install @xterm/addon-search (already in the ecosystem)
  • Add SearchAddon to XTerm.vue, bind Ctrl+F
  • Floating search bar component (reuse spotlight style)
Touch: XTerm.vue, new SearchBar.vue component
quick xterm.js Right-click context menu (copy / paste / clear)

Right-click in any terminal pane shows a small context menu: Copy selection, Paste, Clear, Select All. Warp does this; users expect it. Avoids the "how do I paste?" confusion on first launch.

  • Capture contextmenu on the xterm container, prevent default
  • Show a small positioned menu using the selection-state
  • Paste calls navigator.clipboard.readText() → write_pty
Touch: XTerm.vue, new TerminalContextMenu.vue
quick manaflow Idle-timeout agent rescue

manaflow auto-transitions running → done after 15 s of silence. Rescues tabs where the agent crashed without firing its Stop hook (network drop, OOM, SIGKILL). Currently those tabs stay orange forever until manually Ctrl+C'd.

  • In terminalStatus.ts reducer: after entering running, arm a 30 s inactivity timer
  • Any OSC/hook event resets the timer
  • On expire: call _settle(leaf, ctx) → transitions to done or review
  • Timer exposed via ReducerCtx.setIdleTimer
Touch: terminalStatus.ts, Terminal.vue (ctx implementation)
quick iTerm2 OSC 1337 inline image rendering

iTerm2's inline image protocol: ESC]1337;File=...<base64>BEL. Agents and tools (Python matplotlib, imgcat) already emit it. Render the image inline in the terminal output. Useful when Claude generates charts, diagrams, or screenshots.

  • xterm.js doesn't handle OSC 1337 natively — intercept in the raw data stream before it reaches the terminal
  • Parse payload (width/height hints + base64 blob), inject a <img> via a custom xterm.js ITerminalAddon or overlay
  • Alternatively: strip the sequence and render the image in a floating overlay anchored to the cursor position
Touch: XTerm.vue (new OSC 1337 parser), optional ImageAddon
quick terax-ai Scrollback export to file

Button (or right-click menu item) to save the full terminal scrollback as a .txt or .ansi file. Terax offers this per-session. Useful for sending Claude's output to another agent or reviewing a long run offline.

  • xterm.js: term.buffer.active iterator → collect all lines → join
  • Call Tauri write_text_file or show a save dialog
Touch: XTerm.vue (expose exportScrollback()), Terminal.vue (expose via context menu / toolbar)

Medium Effort — days

medium terax-ai Clickable file paths in terminal output

Scan terminal output for path/to/file.ts:42 patterns and render them as clickable links that open the file in the right panel diff/editor view (or pass to $EDITOR). Terax and Warp both do this. Claude's output is full of file references.

  • xterm.js LinkProvider API: register a regex link provider for [\w./\-]+\.\w+(?::\d+)?
  • On click: resolve path relative to workspace cwd → call open_file Tauri command or emit to right panel
  • Optionally highlight on hover (cursor: pointer)
Touch: XTerm.vue (LinkProvider), lib.rs (open_file command), RightPanel.vue
medium tmux-style Broadcast mode — type to all panes

Toggle a "broadcast" mode on a workspace: keystrokes typed in any focused pane are mirrored to all other panes in the same workspace simultaneously. Classic tmux set-option synchronize-panes on. Useful for running the same command on multiple agents/directories at once.

  • Add a broadcastWs: Set<number> to the UI store
  • In Terminal.vue write_pty path: if broadcast active, write to all leaves' PTYs
  • Visual indicator: orange border around the workspace when broadcast is on
  • Toggle via keyboard shortcut (e.g. ⌘⇧B) or title bar button
Touch: ui.ts store, Terminal.vue, XTerm.vue (emit input up to Terminal)
medium Warp Command blocks — collapsible output regions

OSC 133 A/B/C/D already tells us where commands start and end (just implemented). Use those markers to wrap each command's output in a collapsible "block" UI — click to collapse long npm install output, expand to see it again. Warp's killer feature. Makes long agent sessions much more scannable.

  • Prerequisite: OSC 133 already wired in XTerm.vue ✓
  • Track (row-start, row-end, exit-code) per command in a blocks array
  • Render collapse toggle overlay anchored to the block start row
  • xterm.js doesn't natively support row hiding — need a custom viewport solution or an ITerminalAddon that intercepts rendering
Touch: XTerm.vue (block tracking), new CommandBlock overlay component
medium terax-ai Tab pin / lock

Pin a tab so ⌘W can't accidentally close it and it can't be reordered. Useful for a "main agent" tab you're monitoring while spawning throwaway sub-agent tabs. Terax uses this for its managed sessions.

  • Add pinned?: boolean to Leaf
  • Right-click tab in sidebar → "Pin tab" toggles it
  • Pinned tabs show a pin icon; block close action with a confirmation
  • Persist in SQLite terminal_tabs table
Touch: TerminalSplitView.vue (Leaf), Sidebar.vue, Terminal.vue (block close), lib.rs (persist)
medium Warp Session restore on relaunch

When Burrow relaunches, reattach to still-running PTY sessions (daemon already keeps them alive) AND restore terminal scrollback so the context isn't lost. Currently reattach works but the terminal shows blank until new output arrives.

  • On PTY create in Rust: pipe the last N bytes of the scrollback ring buffer (already tracked?) into the new xterm instance before live data starts
  • Alternatively: use xterm.js serialize addon to snapshot the viewport and restore it on mount
  • Edge case: restore cursor position accurately
Touch: lib.rs (scrollback ring buffer), XTerm.vue (restore snapshot on mount)

Big Effort — week+

big terax-ai Preview pane — embedded localhost webview

Scan terminal stdout for localhost:PORT or 127.0.0.1:PORT patterns. When detected, offer to open an embedded Tauri <webview> in the right panel showing the running app. Automatically refreshes on new output. The single biggest time-saver for web dev agents.

  • Regex on outputBuffer in XTerm.vue → emit devServer(url)
  • RightPanel.vue gets a new "Preview" tab that hosts a Tauri WebviewWindow or <iframe>
  • Auto-detect framework (Vite port 5173, Next.js 3000, Rails 3000 — check common ports)
  • Refresh button + auto-refresh on file change (watch filesystem)
  • Handle HTTPS self-signed certs gracefully
Touch: XTerm.vue, RightPanel.vue, new PreviewPane.vue, tauri.conf.json (webview permissions)
big terax-ai Hunk-by-hunk AI diff review

The git diff panel (right panel) shows diffs per file. Add a "Review" button per diff hunk that fires a one-shot claude -p prompt on that hunk in a sidebar panel. Shows inline comments. Lets you review agent changes file-by-file while the agent is still running another task.

  • Parse unified diff into hunks (already have diff data in git store)
  • Per-hunk "⚡ Review" button → POST to Claude API (not interactive, use -p mode)
  • Stream response into a comment panel below the hunk
  • Cache results so re-opening same diff doesn't re-run
  • Warning: this uses API credits separately from the running agent
Touch: RightPanel.vue (git diff section), new HunkReview component, Tauri HTTP command for API call
big terax-ai Git commit graph (lane-based visualization)

Replace the flat file-list git panel with a lane-based commit graph (like git log --graph or GitLens). Shows branches, merges, and the current HEAD. Especially useful for worktree workflows where agents are committing on multiple branches simultaneously.

  • Parse git log --graph --format=... output into a structured graph
  • Render with SVG lanes (or canvas) in a scrollable list
  • Click a commit → show diff; click a branch label → switch branch
  • Live-refresh when PTY output suggests a commit happened (detect git commit in OSC 133 C/D output)
Touch: git.ts store (new log command), RightPanel.vue, new CommitGraph.vue (SVG)
big Warp Workspace templates

Save the current workspace layout (tabs, agents, split ratios, cwd per pane) as a named template. New workspace → pick a template → instantly gets the same layout. Useful for: "Claude + git panel + preview pane" being one click away instead of rebuilt from scratch every project.

  • Serialize full workspace layout to JSON (tabs → root TreeNode + Leaf metadata)
  • Store templates in SQLite workspace_templates table
  • Template picker in the new-workspace dialog
  • Handle PTY IDs gracefully (new IDs on instantiation, same structure)
Touch: lib.rs (new table + commands), workspace.ts store, Sidebar.vue (new template picker)
